# Консольный симулятор жизни: Модель "Хищник-Жертва"

Простой пошаговый симулятор экосистемы, демонстрирующий взаимодействие живых организмов, механизмы голода, старения и случайных мутаций. Проект написан на языке Python с использованием ООП и соответствует стандартам PEP8.

## Команда и распределение ролей
* **Архитектор проекта (Иванов И.)** — Разработка архитектуры взаимодействия классов, инициализация Git-репозитория, проведение Code Review.
* **Разработчик модуля организмов (Петров П.)** — Создание базового класса Organism и его наследников Prey и Predator.
* **Разработчик модуля экосистемы (Сидоров С.)** — Реализация игрового движка Ecosystem, логики старения популяции и механики случайного размножения.
* **Тестировщик (Кузнецов К.)** — Проверка кода на соответствие PEP8, ручное тестирование граничных сценариев.

## Структура проекта
* `organism.py` — содержит базовую логику существ, их параметры и методы действий.
* `ecosystem.py` — управляет общими списками животных, координирует смену дней и обновляет численность популяции.
* `main.py` — точка входа в программу. Инициализирует мир и запускает цикл симуляции.

## Инструкция по запуску
```bash
python main.py
```

## Процесс разработки в Git
Разработка велась по правилам Git Flow. Каждый модуль разрабатывался в отдельной ветке перед слиянием в main:
* `feature/organism-classes` — добавление классов живых существ.
* `feature/ecosystem-engine` — добавление циклов симуляции дней.
* Все ветки были объединены через систему Pull Request после прохождения проверки кода.
